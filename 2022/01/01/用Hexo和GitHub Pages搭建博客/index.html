搭建博客

发表于 2022-01-01 更新于 2022-01-04 阅读次数：
简单记下搭建博客的流程，方便以后重新搭建博客

Github Pages

Github Pages 其实本身就是 Github 提供的博客服务。 我们在 Github 中创建一个特定格式的 Repository，Github Pages 就会将里面的信息生成一个网页，展示出来。

  1. 注册github账号
  2. 创建仓库
    1. 仓库名必须是类似于username.github.io的形式，设置public，选项都勾选上，然后创建。
    2. 在新建项目的[Settings]->[Options]->[Pages]中选择源分支，然后，选择主题即可。
  3. 简单地编辑一下 README.md 这个文档。
  4. 打开网页 输入博客地址，例如：https://username.github.io就能够看到README.md的内容

Hexo

博客搭建工具，不解释，缺点优点啥的我也不在乎，写个博客自娱自乐。
使用Hexo之前，需要在构筑环境，安装Node.js和git

Node.js的安装

参照我的另外一篇博文

如果对版本没要求的话，直接安装最新版，Node.js安装挺简单的，这边便不赘述了，实在不知道就直接百度好了(菜鸟教程里相关内容写的挺详细的，哈哈)，默认你已安装Node.js。

  1. 查看环境变量Path 查看是否有Node.js安装路径
  2. 打开命令行窗口（vscode终端，cmd,powshell gitbash都行）
  3. 执行npm -v ，出现结果说明安装ok了。

Hexo的安装

  1. 打开命令行窗口（vscode终端，cmd,powshell gitbash都行）

  2. 安装hexo-cli,输入npm install -g hexo-cli

  3. 回车开始安装，有warn不用管，只要没error就行

  4. 输入hexo -v

  5. 最后得到hexo-cli的版本号啥的就ok。

    1. 在Power shell上执行hexo命令，会出现类似于下列情况

      解决办法，用管理员权限打开PowerShell，运行以下命令，最后选择Y回车，就ok了

创建本地博客

  1. 在D盘下创建文件夹 blog
  2. 鼠标右键 blog，选择 Git Bash Here。 如果没有安装 Git，就不会有这个选项。
  3. Git Bash 打开之后，所在的位置就是 blog 这个文件夹的位置。（/d/blog）
  4. 输入 hexo init 将 blog 文件夹初始化成一个博客文件夹。
  5. 输入 npm install 安装依赖包。
  6. 输入 hexo g 生成（generate）网页。 由于我们还没创建任何博客，生成的网页会展示 Hexo 里面自带了一个 Hello World 的博客。
  7. 输入 hexo s 将生成的网页放在了本地服务器（server）。
  8. 浏览器里输入 http://localhost:4000/ 。 就可以看到刚才的成果了。
  9. 回到 Git Bash，按 Ctrl+C 结束。

发布一篇博客

  1. 继续在 Git Bash 里，所在路径还是 /d/blog。输入 hexo new “My First Post”
  2. 在 D:\blog\source_posts 路径下，会有一个 My-First-Post.md 的文件。 编辑这个文件，然后保存。
  3. 回到 Git Bash，输入 hexo g
  4. 输入 hexo s
  5. 前往 http://localhost:4000/ 查看成果。
  6. 回到 Git Bash，按 Ctrl+C 结束。

日常

平时怎样写博客，上传博客，部署博客等等

  1. 使用命令hexo new "博客名"，来生成博客名.md文件。
  2. 选择个你喜欢的编辑器编写博客。
  3. 然后执行hexo g ,生成网页
  4. 执行hexo s，在本地生成下网页，查看下格式啥的，内容啥的
  5. 完事就执行hexo d，pull到github上，其中因为网络不稳定，可能导致上传失败，这种情况就执行hexo clean,清除内容，然后执行hexo d
  6. 如果你的md预览过了，可以直接上传，就直接执行hexo g -d即可，上传失败的话，请执行hexo clean
  7. 上面写的有点绕，主要是因为github，总时不时抽风，push不上去，总结就是有梯子的话，搭个梯子，没有，就自求多福吧。
